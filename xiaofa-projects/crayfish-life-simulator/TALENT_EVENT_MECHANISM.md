# 天赋-事件关联机制 - 核心玩法说明

## 核心设计理念

> 不同天赋开启不同事件，相同天赋提高对应事件概率，让每一局人生都不一样，贴合龙虾转生主题。

---

## 一、天赋系统基础

### 天赋定义结构

```javascript
{
  id: 数字ID,
  name: '天赋名称',
  desc: '天赋描述',
  effectDesc: '属性加成描述',
  level: 0-5 // 天赋等级（0白板→5红色神话），影响抽取概率
  effects: {
    vitality: +2, // 属性加成
    // ... 可以有多个属性加成
    tag: 'tag-name' // 天赋标签，用于事件关联

}
```

## 二、天赋-事件关联机制

### 事件标签定义

每个事件可以带 `tags` 数组：

```javascript
{
  id: 'c4',
  title: '龙虾天赋觉醒',
  minAge: 3,
  maxAge: 5,
  tags: [
    'lobster-regen',
    'lobster-armor',
    // ... 多个天赋标签
  ],
  // ...
}
```

### 筛选规则（game.js `getAvailableEvents`）

```javascript
if (e.tags && e.tags.length > 0) {
  // 天选之子(god)天赋全开，所有匹配标签事件都能触发
  if (!talentTags.includes('god') && 
      !e.tags.some(t => talentTags.includes(t))) {
    return false; // 没有对应天赋，过滤掉
  }
}
// 有至少一个标签匹配，保留事件
```

**规则总结**：
1. 事件没有标签 → 任何天赋都能触发
2. 事件有标签 → 玩家必须至少拥有其中一个标签才能触发
3. `god` 天赋（天选之子） → 所有标签事件都能触发，相当于全开

### 概率加权机制（抽取随机事件）

```javascript
let weight = 1; // 默认权重
if (event.tags) {
  event.tags.forEach(tag => {
    // 玩家有这个标签 → 权重 × 3 → 抽中概率提高3倍
    if (talentTags.includes(tag) || talentTags.includes('god')) {
      weight *= 3;
    }
  });
}
// 按权重放进抽奖池
for (let i = 0; i < weight; i++) {
  weightedNormal.push(event);
}
```

**效果**：
- 有对应天赋的事件，抽中概率是普通事件的 **3倍**
- 多个标签匹配 → 权重叠乘，概率更高
- 保证了天赋能真正影响游戏走向，不同天赋体验不同事件

## 三、完整过滤流程

```
事件筛选顺序：

1. 已触发过的事件 → 过滤掉（不重复触发）
2. 年龄范围筛选 → 不在 minAge ~ maxAge → 过滤掉
3. 属性要求筛选 → 不满足 e.requires 最低属性 → 过滤掉
4. 天赋标签筛选 → 不满足标签要求 → 过滤掉
5. 剩余可用事件 → 按权重随机抽取
```

## 五、核心玩法总结

1. **开局**：玩家分配 25 点基础属性 → 随机抽取 10 个天赋 → 选 3 个带走
2. **游戏中**：每次抽取事件，对应天赋的事件概率提高 3 倍
3. **结果**：不同天赋组合 → 完全不同的事件体验 → 不同结局
4. **重复游玩**：每次天赋不同，体验不同人生，可玩性高

## 六、后续扩展指南

### 新增天赋
1. 在 `index.js` 的 `talentPool` 数组添加天赋定义
2. 给天赋分配标签 `effects: { ... tag: 'tag-name' }`
3. 在事件中添加对应标签 `tags: ['tag-name']`
4. 自动生效，无需修改代码

### 新增标签关联事件
1. 在 `events.js` 添加事件
2. 添加 `tags: ['tag-name']` 对应天赋标签
3. 自动过滤和加权，无需修改核心代码
