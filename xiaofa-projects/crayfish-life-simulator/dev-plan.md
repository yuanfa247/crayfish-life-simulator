# 《养一只小龙虾》15天开发计划

## 项目基本信息

- **定位**：微信小程序开发者
- **唯一项目**：文字人生模拟器《养一只小龙虾》
- **核心玩法**：点击选择→触发剧情→属性变化→多结局，单局5–15分钟
- **最终目标**：15天内完成可上线、带广告联盟变现、带完整开发文档的小程序
- **每日学习**：20分钟+拓展
- **规则**：
    1. 严格按当日任务执行，完成再拓展
    2. 全程只做人生模拟器+小龙虾养成+广告变现
    3. 每日输出学习日报
    4. 同步写开发文档，结构清晰、变量/函数/逻辑全注释，方便其他龙虾接手维护
    5. 必须接入Banner+激励视频广告实现盈利

## 核心玩法

- **开局**：随机生成小龙虾初始属性（活力、智力、财富、运气、颜值）
- **流程**：年龄增长→触发事件→点击选择→属性变化→进入下一年
- **单局**：从幼儿到寿终，5–15分钟一局
- **核心**：每一次点击选择直接影响结局（打工/摸鱼、学习/摆烂、社交/独处、冒险/安稳）
- **结局**：几十种人生结局（龙虾首富、街头流浪、美食博主、退休养老、英年早逝等）
- **广告**：看激励视频→复活/属性翻倍/解锁隐藏剧情/快速成长

## 15天开发计划

### 第1天：项目立项+小程序基础结构

- 初始化小程序，配置app.js/app.json/app.wxss
- 设计《养一只小龙虾》人生模拟器整体架构
- 开始写开发文档.md（项目结构、命名规范）

### 第2天：页面规划+主界面

- 创建页面：首页、游戏主界面、属性面板、结局页、设置页
- 配置路由、tabBar
- 完成主界面布局（文字剧情区+选择按钮+属性面板）

### 第3天：核心数据结构设计（人生模拟器灵魂）

- 定义属性：年龄、活力、智力、财富、运气、颜值、状态
- 设计事件结构：事件ID、触发年龄、剧情文本、选项、属性影响、跳转事件
- 文档更新：数据结构说明、事件格式规范

### 第4天：WXML+游戏界面渲染

- 渲染：剧情文本、选项按钮、属性实时面板
- 实现：点击选项→触发剧情→属性变化→刷新界面
- 完成基础交互框架

### 第5天：JS核心逻辑·年龄+事件触发

- 年龄增长机制（每年+1，阶段：幼儿/青年/壮年/老年）
- 按年龄+属性随机触发事件
- 文档更新：核心逻辑（年龄/事件）

### 第6天：选择分支+属性影响（核心玩法）

- 多选项逻辑：不同选择→不同属性变化→不同后续事件
- 实现：选择→计算属性→更新数据→渲染新剧情
- 编写第一批事件库（幼儿阶段）

### 第7天：事件库扩充+青年阶段剧情

- 扩充事件：学习、打工、社交、随机奇遇
- 实现：事件条件触发（属性达标才解锁）
- 文档更新：事件库设计规范

### 第8天：UI重构+天赋系统+大逻辑重构 ✅ 完成

- 首页UI优化：标题缩小，开始按钮放大
- 新增天赋系统：属性固定总点 + 10选3天赋，天赋分等级颜色（蓝/紫/橙/红），和事件强关联
- 新增龙虾转生设定：小龙虾死在菜市场重生为人，保留7个专属前世天赋
- 结局概率重构：60岁以下30%随机结束，60-105岁50%随机结束，105岁强制结束触发修仙线
- 新增事件去重：同一事件不会重复触发
- 新增天赋页返回按钮，优化用户体验
- 新增广告位：游戏页底部固定Banner广告位，首页移除广告
- 修复大量bug：属性按钮错乱、剩余点数负数、滚动背景断层、广告遮挡内容、自动跳转不触发、天赋重复固定、合并冲突语法错误
- **第一阶段核心开发完成**：原计划15天，实际8天完成 ✓
